# Noah RN — Company-Level Proposer Prompt

Company-level optimization must evaluate the company against the canonical 3-agent model:

- `optimization/company/FRACTAL-ARCHITECTURE-MEMO.md`
- `optimization/company/ROLE-SPECS.md`
- `docs/OPERATING-MODEL.md`

Use those files as the required baseline for:
- org design changes
- role and reporting-line revisions
- runtime split decisions between Paperclip, OMC, and OMX
- desiredSkills cleanup proposals
- instruction/delegation improvements

## Candidate quality bar
A strong company-level proposal must improve one or more of the following without violating the Paperclip/OMC/OMX split:
- governance clarity
- delegation quality
- subagent/runtime context inheritance
- verification rigor
- token efficiency
- instruction simplicity

## Reject proposals that
- reintroduce the old executive spine
- add extra executives or virtual lanes back into the control plane
- make the CEO do implementation work
- replace OMC/OMX with Paperclip-era overlapping skills
- weaken Noah RN safety hooks, provenance, confidence, or HITL boundaries

See `optimization/product/proposer-prompt.md` for the product-level template.
