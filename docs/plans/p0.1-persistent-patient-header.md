## Architectural decisions

1. **Route-scoped layout, not global header.** Introduce `PatientChartLayout` as a nested route wrapper for `/Patient/:id/*` using React Router v7 `<Outlet />`. The header lives in the layout, not `AppShell.Header`, so it unmounts cleanly when the user leaves patient context. This matches the OpenEMR context-bleed-avoidance pattern called out in `research/openemr-workflow-prior-art-analysis.md`.
2. **URL is the source of truth for patient context.** No `PatientContext` provider, no global store. The layout fetches the `Patient` once with `useResource` and passes it to `<PatientHeader>` and to child pages via `<Outlet context={{ patient }}>`.
3. **Custom header, not `@medplum/react`'s `PatientHeader`.** Medplum ships a header component, but the roadmap spec lists exact content slots and demands empty-state fallbacks + stable test-ids — cheaper to own it than fight the component config.
4. **Split into slot components with independent fetches.** Demographics comes free from the `Patient` prop; Encounter / Allergies / CodeStatus / Vitals each own their own hook so a slow query never blocks the demographic render.
5. **Fixed header height (88px) with placeholder-while-loading.** The fallback helper renders `"—"` (or `"NKA"` for allergies) in both loading and empty states so layout never shifts and the "never blank" acceptance check holds.
6. **Tabs move into the layout and become URL-driven** (`?tab=vitals`) via `useSearchParams`, so browser back/forward works and tabs persist across refresh.

| Slot | Query |
|---|---|
| Encounter | `Encounter?subject=Patient/{id}&status=in-progress&_sort=-date&_count=1` |
| Allergies | `AllergyIntolerance?patient=Patient/{id}&clinical-status=active` |
| Code status | `Observation?subject=Patient/{id}&code=http://loinc.org\|75320-2&_sort=-date&_count=1` |

## Acceptance-criteria mapping

| Roadmap check | How this plan satisfies it |
|---|---|
| sticky on scroll | `position: sticky; top: 0` on `PatientHeader` root inside layout |
| persists across chart tab changes | Header lives in `PatientChartLayout`; tabs swap via `<Outlet />`, header never unmounts |
| empty-state fields never blank | Every slot passes values through `fallback()`; loading state renders placeholder, not spinner |
| stable test ids | Convention `data-testid="patient-header-{slot}"` locked in as API contract |
